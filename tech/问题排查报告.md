# 问题排查报告 — COSMO 对话系统

> **日期**：2026-07-23 | **版本**：v2.0（基于最新 4 案例双 Agent 测试）
> **基于**：4 案例端到端测试报告 + AgentMental（AAAI 2026）论文 + COSMO 架构分析

## 一、当前问题列表

### 🔴 问题1：评分系统性高估（案例2 +8、案例3 +5）

**现象**：在两个非重度案例中，COSMO 评分引擎均显著高估。案例2（周思语，轻度 8→16，+8）、案例3（陈浩然，中度 13→18，+5）。高估集中在 Q4（精力）、Q5（食欲）、Q7（注意力）、Q8（精神运动）四个条目。

**证据**：
- 案例2 Q8 偏差最大（0→3，+3）：学生描述"写作业慢、收拾书包慢"→ 评分跳到锚点3（几乎每天明显）。实际是初三学业压力导致的轻度迟缓
- 案例2 Q4/Q5/Q7 各 +1："心理累"→ 锚点2（体力耗竭），"早饭不想吃"→ 锚点2（超半数），"不是每节都走神"→ 锚点2（经常）
- 案例3 Q4/Q5/Q6/Q7/Q8 各 +1：从锚点1跳到锚点2，系统性地无法区分"有几天"和"一半以上天数"

**根因分析**：

1. **锚点匹配缺少语境限定词检测**：学生说"经常但并非每天""有时""偶尔"时，评分 Prompt 未识别这些限定条件，将"经常"直接映射到锚点2。「有几天」和「一半以上天数」之间的语义边界，系统无法精确判定

2. **Q8（精神运动改变）是所有条目中最脆弱的一环**：AgentMental 论文（Table 2）已证实 Q8 在 8 条目中 F1 分数倒数第二（28.9），远低于 Q1（38.7）、Q2（55.7）等。论文明确归因："依赖微妙的语言线索，对推理能力要求最高"。COSMO 中 Q8 在案例2 偏差 +3、案例3 偏差 +1，是最不稳定的条目

3. **全局校准未识别"一致偏高"模式**：评分引擎第二阶段（AGu 角色）设计为逐条目比对偏差≥2才写 calibration_note。案例3 中 5 个条目各 +1，单一都不触发校准，但合在一起是系统性漂移。逐条比对无法捕捉"所有条目一致偏高"

**对标 AgentMental**：AgentMental 的 AGu（更新者）**每聊完一个话题就回头校准全部历史分数**——不是等 9 个条目全部完成后再做一次校准，而是在对话过程中持续修正。COSMO 的两阶段评分虽然弥补了"无实时校准"的缺陷（见 cosmo现状.md 第六章），但全局校准的设计逻辑（仅偏差≥2触发）在"一致偏低/偏高1分"的场景下失效。


### 🔴 问题2：评分严重低估（案例4 -6，Q4/Q7 因信息不足被硬置0）

**现象**：案例4（苏雨桐，重度 27→21，-6）。Q4（精力不足）和 Q7（注意力困难）两条目因对话未覆盖被评分引擎强制置0（各-3）。但案例预设中这两个条目均为 3 分——一个重度抑郁学生无疑存在严重的精力问题和注意力困难。

**证据**：
- Q4 预设 3 → 实际 0（-3），理由"对话信息不足，默认0分"
- Q7 预设 3 → 实际 0（-3），理由同上
- 对话中 AI 围绕 Q9 自伤/自杀意念反复深入确认，对话路径被高风险处理"吞噬"了其他维度的探查空间

**根因分析**：

1. **高风险对话路径挤占了其他条目覆盖空间**：案例4 中 AI 对"我在手腕上划过"→"站在阳台上想跳下去"→"弟弟哭了就下来了"做了深度分层确认（频率→方法→意图→近期性→自杀意念）。这个处理质量非常高（测试报告评价"追问有层次、有温度"），但代价是 Q4/Q7 在对话结束时仍处于未覆盖状态

2. **answer_quality 硬钳位的悲剧性后果**：`insufficient` 条目被代码层强制置0（对标评分 Prompt 双重保险）——这个设计防止的是"假阴性"（没问到≠没问题），但在案例4中恰恰制造了假阴性：一个确实存在极度精力和注意力问题的学生，因为对话没问到而得了0分

3. **对话覆盖检查与 `is_done` 的时序矛盾**：当系统检测到 `allCovered` 时 Q4/Q7 仍为 pending，但某种路径下 `is_done` 仍被触发。对话结束时 Q4/Q7 未被标记为 fallback（也没走兜底）

**核心矛盾**：信息不足条目强制置0的规则假设"对话自然推进中不遗漏维度"，但当高风险路径导致维度遗漏时，这条规则反成了假阴性的制造者。


### 🟡 问题3：SYS 块输出率持续偏低（11%~33%）

**现象**：四案例 SYS 块输出率分别为 33%（案例1）、17.6%（案例2）、11.1%（案例3）、~12%（案例4）。平均约 18%，即 **82% 的对话轮次依赖代码层 fallback 做推进决策**。

**证据**：
- 案例3：18 轮对话仅 2 轮有 SYS 块，`detectAnswerVagueness` 未能触发兜底（学生回答"足够具体"但 SYS 缺失）
- 案例2/3/4 的初评分大面积缺失（仅 1~3 条目有初评）
- 但代码层 fallback 在对话推进上基本正常工作——4 个案例均自然完成、无道别循环

**根因分析**：

1. **角色 Prompt 的叙事密度稀释了 SYS 指令**：对话 Prompt（chat-prompt.ts）将 SYS 块要求放在末尾（~1/3 位置），而 Prompt 开头是 ~70 行的叙事性角色设定。DeepSeek 模型在接收高强度角色扮演指令后进入沉浸式状态，末尾的结构化输出要求被当作"可选的内部备注"

2. **DeepSeek 模型对嵌入自然对话的 HTML 注释格式服从度低**：`<!--SYS...SYS-->` 这种 HTML 注释格式在 OpenAI 系模型中服从度较高，但 DeepSeek V4 Pro/Flash 对其服从度显著更低

3. **无代码层纠正机制**：SYS 缺失路径（chat/route.ts 第 451-480 行）仅在 `item_rounds > 0` 时走 fallback，第一轮缺失只是"等待"。系统不会主动要求模型重试或补充 SYS 块

**对标 AgentMental**：AgentMental 使用独立的 AGev（评估智能体）——信息充分性判断是一个独立 API 调用，不是一个"期望"的副产物。关键区别：AgentMental 的评估输出是**保证会发生的**，COSMO 的 SYS 块是**期望发生但不保证的**。AgentMental 消融实验（Table 3）证明去除追问机制后 Kappa 从 79.8 降至 47.2——信息充分性评估的可靠性是系统精度的关键决定因素。


### 🟡 问题4：高风险处理与覆盖完整性的张力（案例4 Q4/Q7 遗漏）

**与问题2 相关但关注不同的维度**：问题2 关注"硬置0导致评分低估"，问题4 关注"为什么对话引擎没覆盖到这些维度"。

**现象**：案例4 中 Q9 高风险确认流程"吞噬"了 Q4/Q7 的覆盖机会。虽然 AI 的自然确认流程在风险处置上表现出色，但对话引擎缺少"在风险处理后回头检查遗漏条目"的机制。

**根因**：当前架构中高风险处理（Q9 确认 + 3 轮缓和）与其他条目覆盖是两条**互斥路径**——一旦进入风险处理流程，其他 pending 条目的推进被暂停。缺少"风险处理完成后，检查并自动触发遗漏条目的兜底补问"这个环节。


### 🟡 问题5：Q8（精神运动改变）是最脆弱的评分条目

**现象**：Q8 在两次测试中都出现了严重偏差——案例2 +3（0→3）、案例3 +1（1→2）。

**根因分析**：

1. **COGNITIVE 层缺失**：评分 Prompt 中 Q8 的评分锚点仅描述了行为层面（"没有变化""有几天轻微→别人也注意到了""几乎每天明显"），缺少区分"情境性行为减慢"和"临床精神运动迟缓"的指导

2. **学生口头描述 vs 临床定义的鸿沟**：学生说"写作业慢""收拾东西慢""说话变慢"——这些在日常生活中常见于压力、疲劳、性格内向，不一定是精神运动改变。但评分 Prompt 没有"排除条件"来过滤这些非临床原因

3. **AgentMental 论文已证实**（Table 2）：Q8 的 F1 仅 28.9，是 8 个条目中最差的。这不仅是 COSMO 的问题，而是自动化心理评估的共性问题——"精神运动改变"高度依赖对行为细微变化的语义理解


### 🟢 问题6：对话节奏常量偏低（MAX_ITEM_ROUNDS = 2 vs AgentMental d=3）

**现象**：当前每个条目最多追问 2 次。AgentMental 论文中 d=3。没有直接证据表明 d=2 导致信息收集不足（因为 SYS 块缺失使代码层无法精确判断信息是否充分），但较低的追问上限意味着复杂条目容错空间更小。

**对标**：AgentMental 论文（Algorithm 1, Line 5: `while j < d`）将 d 设为 3。消融实验中去除追问机制后性能暴跌（Kappa 79.8→47.2），验证了追问深度的关键性。


## 二、解决思路及建议

### 2.1 核心策略：对标 AgentMental，增强而非重构

COSMO 不应模仿 AgentMental 的 4 Agent 架构（成本不可接受，cosmo现状.md 第五章已论证），但应提取其核心工作机制：

| 提取什么 | AgentMental 怎么做的 | COSMO 怎么做到 |
|---------|--------------------|---------------|
| 可靠的信息充分性评估 | 独立 AGev API 调用 | SYS 块输出可靠性增强（P1-1） |
| 五维定向追问 | AGq 按 emotion/frequency/duration/symptom/impact 生成问题 | Prompt 中结构化追问方向 + SYS 缺失时代码层推断 |
| 实时分数校准 | AGu 每话题完成后回头校准 | 评分 Prompt 全局校准增强（P0-1）+ 阶段性复检（P3-3） |
| 跨条目记忆 | 树形记忆结构（根→话题→陈述节点） | item_summaries 增强 + 跨条目模式发现 Prompt（P2-3） |

### 2.2 分优先级建议

#### P0 — 立即修复（影响面大 + 实施成本低）

| 编号 | 针对问题 | 建议 | 来源 | 方案 |
|------|---------|------|------|------|
| P0-1 | 评分系统性高估（问题1） | 评分 Prompt 增加锚点匹配严格度约束 | AgentMental 锚点匹配 | 在评分 Prompt Step 2（严重度判断）中增加明确规则：① 语境限定词检测——"偶尔""有时""不是每天""并非每天""不是每节""不是每次都"→ 应匹配锚点1（有几天），不匹配锚点2（一半以上）；② Q8 排他条件——"学生描述的行为减慢有明确学业/考试归因时，除非同时出现他人注意到明显变慢、且学生自述情绪原因，否则不应超过锚点1"；③ 频率判断必须核实时间范围——"过去两周内出现过多少天"，而非模糊的"经常" |
| P0-2 | 评分严重低估（问题2） | 对话结束前对未覆盖条目强制触发兜底补问 | — | 在 `allCovered` 判定前加入：遍历 9 个条目的 coverage，对于仍为 `pending` 的条目，不标记 done，而是使用 JSON 响应方式依次弹出兜底选择题（与现有 `handleFallbackScore` 逻辑复用）。这样 Q4/Q7 即使未被对话覆盖，至少通过兜底拿到一个分数（而非硬置0） |
| P0-3 | Q8 最脆弱（问题5） | Q8 评分 Prompt 增加"情境归因排除"指导 | AgentMental Table 2 Q8 分析 | 在评分锚点说明中为 Q8 增加："学生在对话中明确将行为减慢关联到学业压力、考试、作业量时，属于情境性反应而非临床精神运动改变。锚点1仅限他人（同学/家人/老师）也注意到的情况。锚点2/3仅限无明确外部原因的持续行为改变" |

#### P1 — 短期推进（影响面大 + 实施成本中）

| 编号 | 针对问题 | 建议 | 来源 | 方案 |
|------|---------|------|------|------|
| P1-1 | SYS 块输出率低（问题3） | 双重策略：Prompt 结构调整 + 代码层检测重试 | AgentMental AGev 可靠性设计 | ① 将 SYS 块输出指令从 Prompt 末尾移到**开头第二段**（紧接角色设定后、"你现在是心理老师"之前），使用显眼格式 `[系统要求 — 每次回复后必须输出]`；② 在 `createRealStreamResponse` 的 `onComplete` 中检测到无 SYS 块时，向 LLM 追加一句 `[系统] 你的上一条回复缺少必需的分析标记（SYS块）。请在回复末尾补充分析标记，格式见对话开头。` 并重试一次。重试失败则走现有 fallback |

#### P2 — 中期优化（影响面中 + 实施成本中）

## 三、总结

四个案例的最新双 Agent 测试揭示了 COSMO 系统评分引擎的两个核心矛盾：

**矛盾一：锚点匹配的粒度失配。** 评分 Prompt 要求 AI 区分"有几天 vs 一半以上天数""偶尔 vs 经常"，但 AI 从自然对话中提取的频率信息本身就是模糊的——学生不会说"我过去两周有 5 天感到情绪低落"，而是说"经常""有时候""不是每天"。当 Prompt 缺乏精确的语境限定词映射规则时，AI 倾向于**高估**（将模糊表达映射到更高的锚点）。

**矛盾二：高风险处理的优先级 vs 覆盖完整性的保障。** 案例4 中 AI 对自伤/自杀意念的深度确认展示了 COSMO 风险处置体系的最佳一面——追问有层次、有温度、判定准确。但这个成功的风险处理恰恰导致 Q4/Q7 被遗漏，最终评分偏低估 6 分。这是一个"做对了风险处置但做错了评分完整性"的讽刺结果。解决方向不是削弱风险处理，而是在风险处理完成后**强制保证**其他条目的覆盖。

AgentMental 论文（Table 3 消融实验）给出了量化的改善预期：追问+记忆模块贡献了 Kappa 从 47.2 到 79.8 的 32.6 个百分点提升。COSMO 当前架构已具备这两者的雏形（SYS 块追问 + item_summaries），但需要提升输出可靠性和记忆关联深度，而不是增加 Agent 数量。

> **文档说明**：本报告基于 2026-07-23 最新 4 案例双 Agent 测试数据编写。问题优先级依据"影响面×可修复性"矩阵排序。解决思路来自 AgentMental（AAAI 2026）、HopeBot（UCL/NHS 2025）、BDI-FS-GPT（JMIR 2026）、MAGI（ACL 2025 Findings）四篇文献和 COSMO 架构分析（cosmo现状.md）。
