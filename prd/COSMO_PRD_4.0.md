# COSMO PRD V4.0

> **版本**：V4.0 | **日期**：2026-07-22 | **基于**：BRD V3.0 + 9 篇文献对比分析 + COSMO 源码
> **阅读对象**：产品经理、开发工程师

---

## 一、产品定位与目标

### 1.1 核心定位

COSMO 是一个面向 K-12 学生的 AI 对话式心理筛查系统，用自然对话替代传统 PHQ-9 自评量表，作为学校心理筛查的第一步。

**一句话：让学生感觉是在聊天，但系统在背后做评估。**

### 1.2 为什么是现在

| 事实 | 来源 |
|------|------|
| AI 对话 PHQ-9 评分与自评 ICC = 0.92 | HopeBot（UCL/NHS, 2025） |
| AI 对话评分的临床一致性优于自评（κ = 0.72 vs 0.55） | BDI-FS-GPT（JMIR, 2026） |
| 自适应追问是准确性的关键因子——去掉追问 Kappa 降幅 ~24% | AgentMental（AAAI, 2026） |
| 结构化记忆对准确性的贡献与追问同等重要——去掉记忆 Kappa 降幅 ~26% | AgentMental（AAAI, 2026） |
| 用户接受度优于传统方式——触达率是表单的 2.5 倍 | Perla（Madrid, 2020） |

**结论：对话式筛查的科学可行性在成人群体中已经有充分证据。K-12 场景是空白——这正是 COSMO 要做的事。**

### 1.3 成功标准

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 评分一致性 | AI 对话评分 vs 自评 PHQ-9 的 ICC ≥ 0.85 | 试点组内对照 |
| 安全性 | 零风险漏判（所有高风险个案被正确识别） | 试点全量复核 |
| 覆盖面 | 9/9 条目全部获取有效信息（回答 or 兜底） | 系统内置：coverage 状态机 |

## 二、需求与可行性

### 2.1 K-12 场景的特殊性

学术研究全部基于成人，K-12 场景有三个关键差异：

| 维度 | 成人研究场景 | 学校场景 | 产品应对 |
|------|------------|---------|---------|
| 表达能力 | 语言成熟、逻辑清晰 | 低年级学生可能说不清感受 | 角色用语降维 + 兜底机制确保不遗漏 |
| 参与动机 | 自愿报名、有报酬 | 可能有"被安排感"、敷衍回答 | 对话自然度 > 评估感；去病理化表达 |
| 防御心理 | 较低 | 担心"被贴标签"、"被叫家长" | AI 角色不评判、不下诊断、不贴标签 |

### 2.2 产品不做的事

- ❌ **不诊断**：COSMO 输出 PHQ-9 分数，不输出 DSM-5 诊断结论
- ❌ **不干预**：不提供心理治疗或危机干预，仅提供资源引导
- ❌ **不替代人**：筛查结果供心理老师复核，最终判断和责任在专业人
- ❌ **不越界收集**：不主动询问非量表维度（家庭、童年经历等）

## 三、产品逻辑

### 3.1 核心设计理念：「结构化的不确定性管理」

**根本矛盾：** 对话必须自然流畅（让学生愿意多说），同时又必须严谨可控（确保每个条目都获取了有效信息）。

**解决思路：** 不让 LLM 做决策。LLM 负责「理解学生说了什么」，代码层负责「接下来该做什么」。

```
学生说话 → LLM 理解语义 + 分析信息充分度
                    ↓
          代码层统一决策：
          信息充分 → 推进下一条目
          信息不足 → 追问（告诉 LLM 追问什么维度）
          追问后仍不足 → forced-choice 二选一
          仍无法确定 → 硬兜底（PHQ-9 原题 + 四选一）
```

这个思路对应 BRD 的核心命题——「AI 负责灵活性，代码负责确定性」。

### 3.2 对话生命周期

```
破冰（2 轮）
  └→ 自然问候，往轻松方向聊，不急着问正事
      └→ 访谈（Q1 → Q9，逐一覆盖）
            ├─ 首轮：AI 自由提问（自然聊天方式）
            ├─ 追问：信息不足时，AI 针对缺失维度追问（最多 2 次）
            ├─ forced-choice：追问后仍不足，二选一
            └─ 硬兜底：仍无法确定，展示原题 + 4 个频率选项
      └→ 收尾（全部覆盖后）
            └→ 分级收尾语 + 报告生成入口
```

**关键约束：**
- 每个条目必须通过「自然充分 / forced-choice / 硬兜底」三条路径之一，不可跳过
- 对话结束的唯一条件是 9 个条目全部覆盖（answered 或 fallback）
- 无时间限制——推进速度由信息充分度决定，不由墙钟决定

### 3.3 三层安全网

```
第一层：AI 自评（SYS 块）
  └→ 每轮回复后 LLM 输出内部分析：info_sufficiency + item_score_guess + risk_confirmed

第二层：代码层决策
  └→ 解析 SYS 块 → 统一决策（推进/追问/兜底）
  └→ SYS 解析失败时 → detectAnswerVagueness() 代码层兜底判断

第三层：硬兜底（最后一层）
  └→ 自然对话无法获取信息时 → 直接展示 PHQ-9 原题让用户选择
  └→ 保证：零信息丢失——"没问到 ≠ 没问题"
```

### 3.4 评分逻辑

评分分为两个阶段——第一阶段嵌入对话中，第二阶段在对话完成后独立进行。整体对标 AgentMental 的 AGs→AGu 两阶段架构。

**第一阶段 — 对话中逐项初评（AGs 角色）：**
每轮对话的 SYS 分析块中包含 `item_score_guess`（0-3/-1），AI 基于当前条目的单段对话给出即时推测。条目推进时，代码层将初评分写入 `session.item_scores_initial`，供第二阶段交叉校验。

**第二阶段 — 对话后全局校准（AGu 角色）：**
对话完成后，评分引擎读取完整对话记录，独立进行 PsyCoT 三步推理，并与第一阶段的初评分交叉校验。全局校准阶段可以看到全部 9 条目的对话上下文，发现并修正逐项初评可能存在的偏差。

**PsyCoT 三步推理（对标 MAGI）：**
1. **症状锚定**：从对话中提取该条目的具体事实（学生说了什么）
2. **严重度判断**：综合频率、强度、影响范围
3. **锚点匹配**：匹配到 PHQ-9 标准评分锚点（0/1/2/3），输出 `matched_anchor_index`

**全局校准规则：**
- 感知对话整体的情绪基调和功能状态，形成整体印象
- 识别与整体印象明显矛盾的条目（偏离幅度 ≥ 2 分才修正，1 分差异属正常浮动）
- 修正时填写 `calibration_note` 说明原因，汇总为 `calibration_summary`
- 条目之间相互独立——不得以"某个维度分数高所以其他维度也该高"为由调整

**双保险：**
- LLM 层：prompt 要求信息不足时默认 0 分，不做猜测
- 代码层：`item_answer_quality === 'insufficient'` 的条目强制钳位为 0 分，覆盖 LLM 输出

**Q9 硬规则：** Q9 任何非零得分 → risk_level 强制 severe，不受总分阈值影响。

## 四、产品架构

```
┌──────────────────────────────────────────────┐
│                  对话引擎                      │
│  ┌─────────┐  ┌──────────┐  ┌─────────────┐  │
│  │ 角色 Prompt │  │ SYS 分析块 │  │ 代码层决策树  │  │
│  │ (自然聊天)  │  │ (LLM 自评) │  │ (推进/追问/兜底)│  │
│  └─────────┘  └──────────┘  └─────────────┘  │
│                      ↓                        │
│           覆盖状态机 + 条目摘要记忆              │
└──────────────────────────────────────────────┘
         ↓ 会话完成               ↘ item_scores_initial（逐项初评）
┌──────────────────────────────────────────────┐
│                  评分引擎                      │
│  ┌──────────────────┐  ┌──────────────────┐  │
│  │ PsyCoT 三步推理    │  │ 信息不足硬钳位     │  │
│  │ (症状→严重度→锚点) │  │ (代码层覆盖 LLM)  │  │
│  └──────────────────┘  └──────────────────┘  │
│  ┌──────────────────────────────────────┐    │
│  │ 全局校准（交叉校验初评分，修正偏差）     │    │
│  └──────────────────────────────────────┘    │
└──────────────────────────────────────────────┘
                       ↓ 评分完成
┌──────────────────────────────────────────────┐
│                  报告引擎                      │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │ 状态凝练句 │  │ 此刻的你  │  │ 往前一步   │  │
│  │ (10-15字) │  │ (三层递进) │  │ (分层建议) │  │
│  └──────────┘  └──────────┘  └───────────┘  │
└──────────────────────────────────────────────┘
```

### 各引擎职责与设计依据

| 引擎 | 职责 | AI 做什么 | 代码层做什么 | 设计依据 |
|------|------|----------|------------|---------|
| **对话引擎** | 逐一覆盖 9 个 PHQ-9 维度，逐项初评 | 自由生成自然对话 + 每轮输出 SYS 自评块（含 `item_score_guess`） | 解析 SYS 块，统一决策（推进/追问/兜底），管理覆盖状态机，写入 `item_scores_initial` | AgentMental 双参数控制 + MAGI 节点守卫 + COSMO 现有 SYS 块 |
| **评分引擎** | 对话结束后独立评分 + 全局校准 | PsyCoT 三步推理（锚定→判断→匹配）+ 交叉校验初评分，修正偏差 | insufficient 钳位 + matched_anchor_index 校验 + 评分结果缓存 | MAGI PsyCoT + HopeBot/BDI-FS-GPT 评分解耦 + AgentMental AGs→AGu |
| **报告引擎** | 生成学生可读的反馈 | 生成凝练句 + 状态分析 + 分层建议 | 病理化词汇过滤 + 长度截断 + 兜底模板 | COSMO 现有，无文献对标 |

### 安全模块（贯穿所有引擎）

| 模块 | 位置 | 触发方式 | 行为 |
|------|------|---------|------|
| 风险检测 | 对话引擎 | 关键词+正则，每次用户消息 | 自然确认（不弹窗）→ 二次确认 → Q9 兜底 |
| 后处理过滤 | 对话引擎 | AI 回复生成后 | 移除病理化词汇（抑郁/焦虑/障碍/症状/诊断） |
| 收尾语分级 | 对话引擎 | 9 条目全部覆盖 | 无风险 / 有风险 / 自杀意念 → 三类差异化话术 |

## 五、V4.0 核心变更（相对于现状 Demo）

### 变更一览

| 模块 | 现状 | V4.0 | 依据 |
|------|------|------|------|
| **对话控制** | 墙钟时间（12分钟提醒/15分钟强制收尾） | 轮次驱动：每条目最多2次追问，信息充分即推进 | AgentMental（无文献用墙钟控制） |
| **信息充分性** | 二值（sufficient/insufficient） | 三级评分（0=严重不足/1=部分/2=充分） | AgentMental（0-2 necessity score） |
| **追问深度** | 最多1次追问 | 最多2次追问 + forced-choice 中间层 | AgentMental（d参数）+ MAGI（forced-choice） |
| **追问方向** | 无方向引导 | 每轮输出 missing_dimensions，下轮针对性追问 | AgentMental（五维信息提取） |
| **评分架构** | 对话后一次性评分，无跨条目校验 | 两阶段：对话中逐项初评(AGs) + 对话后全局校准(AGu)，利用全部 9 条目上下文修正偏差 | AgentMental AGs→AGu |
| **评分推理** | 四步判断法 | PsyCoT 三步推理（症状锚定→严重度→锚点匹配） | MAGI PsyCoT |
| **评分解耦** | LLM 直接输出 0-3 分数 | LLM 输出锚点索引 + 代码层校验一致性 | HopeBot + BDI-FS-GPT |
| **评分安全** | prompt 层提醒不猜测 | prompt + 代码层双保险 insufficient 条目光钳位 0 | COSMO 现有 Q9 逻辑推广 |
| **全局校准** | 无 | 感知整体情绪基调、识别异常条目（偏离≥2分才修正）、输出 calibration_note+calibration_summary | AgentMental inter-topic correlation |
| **条目记忆** | 无跨条目记忆 | 每个条目完成后输出结构化摘要，供后续条目上下文感知 | AgentMental（树形记忆 + 主题节点 summary） |

### 不变的部分

- ✅ 角色设定（学校心理老师 + 叙事性自画像）
- ✅ SYS 分析块机制（LLM 每轮自评，代码层解析）
- ✅ 硬兜底机制（直接展示 PHQ-9 原题 + 四选项）
- ✅ 风险检测与自然对话处置（不弹窗）
- ✅ 后处理过滤（病理化词汇）
- ✅ 报告三模块结构（凝练句 + 此刻的你 + 往前一步）
- ✅ 覆盖状态机驱动的 done 判定
